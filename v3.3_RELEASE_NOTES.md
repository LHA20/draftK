## What's New

### 🔌 Arduino/ESP32 Serial Port Integration

The Parking Management System now seamlessly integrates with Arduino and ESP32 microcontrollers for fully automated RFID-based parking operations.

#### Key Features

1. **Auto-detect Serial Ports**
   - Automatically discovers connected USB devices
   - Shows device names and descriptions
   - Refresh button to rescan for new devices
   - Works on Windows, macOS, and Linux

2. **Configurable Baud Rates**
   - Support for 9600, 19200, 38400, 57600, 115200
   - Pre-selected before connection
   - Change requires disconnect/reconnect

3. **JSON Card Scan Processing**
   - Parses standard JSON from Arduino/ESP32
   - Extracts card ID, direction, and slot
   - Validates message format
   - Robust error handling

4. **Automatic License Plate Capture**
   - Triggered on card scan (300ms delay)
   - Uses existing YOLO detection
   - Saves full frame and cropped image
   - Stores to captured_images folder

5. **Smart Check-in/Check-out**
   - Determines operation from "gate" field
   - ENTRY → Auto check-in
   - EXIT → Auto check-out
   - Calculates fees automatically

#### UI Updates

New **🔌 ARDUINO/ESP32 CONFIGURATION** panel with:
- Port selection dropdown
- Port refresh button (🔄)
- Baud rate selection
- Connection status indicator (🟢/🔴)
- Connect/Disconnect button

## JSON Message Format

### Card Scan Event (Required)

```json
{
  "event": "CARD_SCAN",
  "uid": "A1B2C3D4",
  "gate": "ENTRY",
  "slot": 1
}
```

**Transmission:** JSON string + newline character (\\n)

**Fields:**
- `event` (string): "CARD_SCAN" (case-sensitive)
- `uid` (string): Unique card ID (4-32 chars)
- `gate` (string): "ENTRY" or "EXIT" (case-sensitive)
- `slot` (integer): Parking slot number (1-100)

## Known Limitations

1. **Single port connection** (one gate at a time)
   - Future: Multi-gate support planned

2. **No acknowledgment** (Arduino doesn't receive feedback)
   - Future: ACK mechanism planned

3. **Plain text JSON** (no encryption)
   - Future: Encryption support planned

4. **Message loss if processing** (no queue)
   - Future: Buffer queue planned

These are by design for initial release and will be addressed in v3.4+

## Troubleshooting

### Port Not Detected
- Check USB cable connection
- Verify device drivers installed
- Click refresh button (🔄)
- Try different USB port

### Connection Failed
- Check baud rate matches device
- Verify port not used elsewhere
- Check device is powered on
- Try restarting device

### License Plate Not Captured
- Check camera is working (manual test first)
- Verify lighting is adequate
- Ensure YOLO model loaded
- Check console for error messages
